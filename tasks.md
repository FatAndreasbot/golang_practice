Реализуйте структуру `CircuitBreaker`,:

### 1. Состояния
- **Closed**:
    - Выполняются все HTTP-запросы
    - При 3 последовательных ошибках → переход в Open
- **Open**:
    - Все запросы сразу возвращают `ErrCircuitOpen`
    - Через 10 секунд → переход в Half-Open
- **Half-Open**:
    - Разрешается **только 1 пробный запрос**
    - Успех → переход в Closed, сброс счетчика
    - Ошибка → возврат в Open на 10 секунд

### 2. Критерии ошибки (триггеры для счетчика):
- HTTP статус ≥ 500
- Таймаут запроса (используйте `context.WithTimeout`)
- Любая ошибка сети (DNS, connection refused и т.д.)

# Реализация PubSub менеджера 

Простой менеджер публикации/подписки (PubSub) для организации асинхронной коммуникации между компонентами системы.

## Требования
- Публикация сообщений в каналы по идентификатору
- Подписка на каналы с получением сообщений
- Потокобезопасность при работе с подписчиками
- Буферизация сообщений для предотвращения блокировок
- Гарантия доставки сообщений всем подписчикам

1. **Лимитер попыток:**
- Максимум 5 попыток входа за 1 минуту.
- Восстановление 1 попытки каждые 12 секунд при отсутствии активности пользователя.
В системе реализован механизм, который отслеживает количество попыток входа и ограничивает их, если пользователь превышает допустимый лимит.

2. **Функциональность:**
- Для каждого пользователя (по ID) создается отдельный бакет (контейнер токенов).
- Метод `AllowLogin(userID string) error` проверяет возможность входа пользователя, принимая в расчёт количество доступных токенов в бакете.
- Автоматический дренаж (очистка) токенов в фоновом режиме — токены восстанавливаются по мере времени.

func main() {

    x[0]++

    x = append(x, 100)
    x[0]++

f3 := func(x *[]int) {
    (*x)[0]++

f4 := func(x *[]int) {
    *x = append(*x, 100)
    (*x)[0]++
